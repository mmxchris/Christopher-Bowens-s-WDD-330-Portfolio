Week 01 Notes

Build for mobile 1st.

  • Much easier to design for mobile 1st then is to retrofit a desktop version for mobile.
  • Desktop sales are declining
  • People don’t walk around with their heads in a laptop
  • Start small and scale up
  • Be there when customers need you. Be useful to users quickly. Be quick, less steps in a process.
  • Reachability matters – smart phone users mostly use their thumps with a one-handed grip, with tablet they use both thumbs with a 2 handed grip.

    • Speed matters

  • longer the load time the lower the traffic.
  • Network can also affect traffic and speed.
  • Reduce image usage if you need images use SVG/Webfont
  • Optimize CSS and JS files
  • GZIP to reduce file size.

    • Strategic Menu Design

  • Commonly used menu items at the bottom for easier access.
  • Perception matters
  • Steller experience from the git go

    • Designing Intuitive and Mobile Friendly Forms

  • Reduce in forum inputs, put labels in input fields, larger input fields, use float labels, larger check boxes, only ask for information needed, break large forums into section, add a progress, enable autofill
  • Don’t hide password input, hiding it can cause error which can lock out accounts and cause support cost
  • Numbers matter
  • Calls matter
  • Use right keyboard for input
  • Calls matter
  • Use anchor tag so users can just click on the number to call

    • Designing a Better Mobile Advertising User Experience

  • NO POP UP! Or in between ads, small simple ads in content, user don’t go to a web site for ads, content should come first.
